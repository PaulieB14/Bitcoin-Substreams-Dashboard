Technical Implementation Details
Data Processing Pipeline
  1. Data Extraction: Bitcoin blockchain data is extracted using Substreams technology, which provides efficient streaming of blockchain data.
  2. Data Transformation: The raw blockchain data is processed and transformed into structured formats using the following modules:
    • map_block_meta: Extracts metadata from each block including height, timestamp, and hash.
    • map_transactions: Extracts detailed transaction data including inputs, outputs, and values.
  3. Data Storage: The processed data is stored in Parquet format, which provides efficient compression and columnar storage for analytics.
  4. Data Analysis: Python scripts using pandas and matplotlib analyze the data and generate visualizations.
Running the Pipeline

To update the dashboard with new blockchain data:

cd scripts && ./run-substreams-to-parquet.sh

This script will:

  • Stream blockchain data from the Substreams API
  • Convert the data to Parquet format
  • Generate updated visualizations
Customization Options

The block range and other parameters can be customized by editing the scripts/run-substreams-to-parquet.sh script:

  • Adjust START_BLOCK and END_BLOCK to analyze different time periods
  • Modify TX_START_BLOCK and TX_END_BLOCK to control transaction data volume
  • Edit queries/analyze_bitcoin_data.py to create custom visualizations
